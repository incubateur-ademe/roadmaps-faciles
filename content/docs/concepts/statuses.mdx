title: Statuts
description: Les statuts permettent de suivre l'avancement des contributions dans votre workflow.
---

Les **statuts** représentent les étapes de votre workflow produit. Ils permettent de communiquer visuellement l'état d'avancement de chaque contribution à vos utilisateurs.

## Exemples de statuts

Un workflow typique pourrait inclure :

- À étudier
- Planifié
- En cours de développement
- Terminé
- Refusé

Vous êtes libre de définir autant de statuts que nécessaire et de les nommer comme vous le souhaitez.

## Propriétés d'un statut

Chaque statut possède trois propriétés :

| Propriété | Description |
|-----------|-------------|
| **Nom** | Le libellé affiché sur les contributions (ex. : "En cours"). |
| **Couleur** | Une couleur choisie parmi la palette du système de design (DSFR) pour distinguer visuellement les statuts. |
| **Visibilité roadmap** | Indique si les contributions avec ce statut apparaissent dans la feuille de route. |

## Statuts et feuille de route

La **feuille de route** (roadmap) est une vue Kanban qui organise les contributions par colonnes de statut. Seuls les statuts marqués comme "visibles dans la roadmap" y apparaissent.

Cela permet de montrer publiquement l'avancement de votre produit tout en gardant certains statuts internes (ex. : "Refusé" ou "À trier") hors de la vue publique.

## Ordonnancement

Les statuts peuvent être réordonnés pour refléter la progression naturelle de votre workflow. L'ordre défini dans l'administration est utilisé pour :
- Les colonnes de la feuille de route
- L'affichage des badges de statut

La gestion des statuts se fait dans [Administration > Statuts](/doc/admin/statuses-management).
