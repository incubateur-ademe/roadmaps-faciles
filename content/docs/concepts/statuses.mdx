---
title: Statuts
description: Les statuts permettent de suivre l'avancement des posts dans votre workflow.
---

Les **statuts** représentent les étapes de votre workflow produit. Ils permettent de communiquer visuellement l'état d'avancement de chaque post à vos utilisateurs.

## Exemples de statuts

Un workflow typique pourrait inclure :

- À étudier
- Planifié
- En cours de développement
- Terminé
- Refusé

## Configuration

Chaque statut possède :

- **Nom** — Le libellé affiché (ex. : "En cours").
- **Couleur** — Une couleur du système de design (palette DSFR) pour distinguer visuellement les statuts.
- **Visibilité roadmap** — Indique si les posts avec ce statut apparaissent dans la vue roadmap.

## Roadmap

La **roadmap** est une vue Kanban qui organise les posts par colonnes de statut. Seuls les statuts marqués comme "visibles dans la roadmap" y apparaissent.

Cela permet de montrer publiquement l'avancement de votre produit tout en gardant certains statuts internes (ex. : "Refusé" ou "À trier") hors de la vue publique.

## Ordonnancement

Les statuts peuvent être réordonnés pour refléter la progression naturelle de votre workflow. L'ordre défini dans l'administration est utilisé pour les colonnes de la roadmap.
