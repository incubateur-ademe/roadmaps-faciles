Un **tableau** (board) est un espace thématique au sein d'un espace de travail. Il permet de catégoriser les retours utilisateurs par sujet.

## Exemples de tableaux

- "Suggestions" — Idées d'amélioration
- "Bugs" — Problèmes remontés par les utilisateurs
- "Améliorations" — Propositions d'évolution

## Caractéristiques

- **Titre et description** — Chaque tableau possède un nom et une description optionnelle. La description supporte le format **Markdown**, ce qui permet d'y ajouter des liens, de la mise en forme ou des instructions.
- **Ordonnancement** — Les tableaux peuvent être réorganisés pour contrôler leur ordre d'affichage dans l'interface.
- **Tableau principal** — Un tableau peut être désigné comme page d'accueil de l'espace. C'est celui qui s'affiche quand un utilisateur arrive sur votre espace.

## Vues disponibles

Les contributions d'un tableau peuvent être affichées selon deux modes, accessibles via les boutons en haut à droite du tableau :

### Vue cartes

Affichage en grille avec aperçu détaillé de chaque contribution : titre, description tronquée, compteur de votes, statut, tags et auteur.

![Vue cartes d'un tableau](/doc/screenshots/board-cards.png)

### Vue liste compacte

Affichage condensé, inspiré du style GitHub Issues. Chaque contribution apparaît sur une seule ligne avec le compteur de votes, le statut, le titre, l'auteur, la date et les tags. Idéal pour parcourir rapidement un grand nombre de contributions.

![Vue liste d'un tableau](/doc/screenshots/board-list.png)

## Tri

Les contributions peuvent être triées par :

| Tri | Description |
|-----|-------------|
| **Tendances** | Contributions les plus actives récemment (votes, commentaires). C'est le tri par défaut. |
| **Top** | Contributions avec le plus de votes, tous temps confondus. |
| **Récents** | Contributions les plus récemment créées. |
