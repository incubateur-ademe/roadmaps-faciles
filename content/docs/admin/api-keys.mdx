Les **clés API** permettent un accès programmatique à votre espace de travail pour des intégrations ou des automatisations.

![Gestion des clés API](/doc/screenshots/admin-api-keys.png)

## Créer une clé API

1. Rendez-vous dans **Administration > API**.
2. Cliquez sur **Créer une clé API**.
3. Donnez un **nom descriptif** à votre clé (ex. : "Intégration Slack", "Script de reporting").
4. Validez la création.

<Callout type="warn">
  La clé API complète n'est affichée **qu'une seule fois** à la création. Copiez-la immédiatement et conservez-la en lieu sûr. Il n'est pas possible de la retrouver ultérieurement.
</Callout>

## Gérer les clés

La liste affiche vos clés existantes avec :

| Information | Description |
|-------------|-------------|
| **Nom** | Le nom descriptif de la clé. |
| **Préfixe** | Les premiers caractères de la clé, pour l'identifier. |
| **Date de création** | La date à laquelle la clé a été générée. |

## Supprimer une clé

La suppression d'une clé est **immédiate et irréversible**. Toute intégration utilisant cette clé cessera de fonctionner dès la suppression.

Avant de supprimer une clé, vérifiez qu'aucune intégration active ne l'utilise.

<Callout type="info" title="Section technique">
La section ci-dessous concerne l'utilisation technique des clés API.
</Callout>

## Sécurité des clés

Les clés API sont stockées de manière sécurisée :

- Seul un **condensé cryptographique** (hash) de la clé est stocké en base de données — la clé en clair n'est jamais conservée.
- Chaque clé est composée d'un **préfixe commun** à l'espace et d'une partie **aléatoire** unique.
- Les clés sont associées à un espace de travail spécifique et ne donnent accès qu'aux données de cet espace.
