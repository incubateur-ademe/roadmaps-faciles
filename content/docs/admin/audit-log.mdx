Le **journal d'audit** enregistre toutes les actions significatives effectuées sur votre espace de travail, à des fins de traçabilité et de conformité. Accédez-y via **Administration > Journal d'audit**.

<ImageWithTheme srcLight="/doc/screenshots/admin-audit-log-light.png" srcDark="/doc/screenshots/admin-audit-log-dark.png" alt="Journal d'audit" />

## Informations enregistrées

Chaque entrée du journal contient :

| Information | Description |
|-------------|-------------|
| **Action** | Le type d'action effectuée (ex. : création de tableau, changement de rôle, approbation de contribution). |
| **Utilisateur** | La personne qui a réalisé l'action. |
| **Date et heure** | Le moment exact de l'action. |
| **Résultat** | Succès ou erreur. |
| **Cible** | La ressource concernée (contribution, tableau, membre...). |
| **Adresse IP** | L'adresse IP de la personne ayant effectué l'action. |

## Filtrer les entrées

Utilisez les filtres disponibles pour affiner votre recherche :

- **Type d'action** — Filtrez par catégorie (ex. : uniquement les actions sur les membres, ou sur les contributions).
- **Période** — Sélectionnez une plage de dates pour cibler une période précise.

## Types d'actions auditées

Le journal couvre l'ensemble des actions d'administration :

| Catégorie | Exemples d'actions |
|-----------|-------------------|
| **Tableaux** | Création, modification, suppression, réordonnancement |
| **Contributions** | Création, approbation, rejet, suppression, modification, changement de statut |
| **Membres** | Invitation, changement de rôle, blocage, suppression |
| **Paramètres** | Modification des paramètres généraux, authentification, sécurité, feuille de route |
| **Domaines** | Modification du domaine personnalisé |
| **Clés API** | Création, suppression |
| **Webhooks** | Création, suppression |
| **Espace** | Purge des données, suppression de l'espace |

## Export CSV

Vous pouvez exporter le journal d'audit au format **CSV** pour un traitement externe. Le bouton d'export génère un fichier contenant toutes les entrées filtrées.

## Conservation des données

Les entrées du journal d'audit sont conservées **indéfiniment**. Elles survivent à la suppression des utilisateurs ou des ressources concernées — si un membre est retiré de l'espace, ses actions passées restent visibles dans le journal.

<Callout type="info" title="Section technique">
Chaque entrée du journal inclut un **identifiant de corrélation** (correlation ID) qui permet de relier une action à une requête HTTP spécifique. Cet identifiant est utile pour le diagnostic en cas d'erreur, en complément des logs serveur.
</Callout>
